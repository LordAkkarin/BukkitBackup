Thanks for choosing the Evil-Co Backup Plugin! When installing the plugin everything is already set-up for you unless you need to set up an external storage.

To install this plugin you will need to extract the distribution archive into your plugin folder (including it's dependencies inside the Backup folder; you may omit the readme and license files if you wish). Please do not modify the directory structure. The plugin expects all dependencies to be inside the Backup/lib directory.

Advanced Configuration

Apart from the default configuration (backup to local filesystem) you are able to store your backups on Amazon Glacier (a low-cost solution for long-term data storage). To do so you will need to change a few properties in the configuration.xml:

...
<StorageMethod>AMAZON_GLACIER</StorageMethod>
<AmazonAccessKeyID>YourAccessKey</AmazonAccessKeyID>
<AmazonAccessSecretKey>YourSecretKey</AmazonAccessSecretKey>
<AmazonEndpoint>us-east-1</AmazonEndpoint>
<AmazonBucket>YourVault</AmazonBucket>
...
  • StorageMethod Can be NONE or AMAZON_GLACIER depending on your remote storage.
  • AmazonAccessKey Your Amazon Access Key (see AWS Documentation; only needed if using Glacier storage.
  • AmazonAccessSecretKey Your Amazon Access Key (see above).
  • AmazonEndpoint Your endpoint (e.g. us-west-1, us-west-2, us-east-1, etc). A full list of all endpoints is available here.
  • AmazonBucket Your bucket/vault name.

Common Errors

  • Could not backup file worldedit.log.lck

    This error might occur if the plugin is used together with WorldEdit on Windows systems due to the file lock system of Windows. This error is not important and will not cause any trouble.
